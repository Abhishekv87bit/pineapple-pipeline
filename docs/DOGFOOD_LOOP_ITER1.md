## What Still Needs Fixing for Iteration 2

### Critical: Stubs block real code (ROOT CAUSE of all 4 review criticals)

The Setup stage scaffolds empty stubs into `src/kfs_manifest/` and `tests/`. The Builder then generates real code but targets those SAME paths. Since the stubs exist and are non-empty (77 bytes), the SKIP protection blocks the real content. The Builder's LLM then tries alternative paths (`kfs_core/`, `kfs_cli/`) which creates module duplication.

**Fix needed:** The stub-detection fix (checking for "generated by Pineapple Setup") was applied mid-run but the pipeline had already entered retry. For iteration 2, this fix is already in place and should work on a clean run.

### Critical: Builder writes to wrong paths

Even when stubs are detected, the builder LLM sometimes writes to `kfs_core/` or `kfs_cli/` instead of the planned `src/kfs_manifest/` path. This creates duplicate module hierarchies.

**Fix needed:** Add path enforcement in the builder prompt -- remind the LLM to use ONLY the paths listed in `files_to_create` / `files_to_modify`, not invent new paths.

### Critical: Schema drift between tasks

`kfs_core/io.py` imports `Transformation` from `kfs_core.models` but that class doesn't exist in the actual models. `kfs_cli/cli.py` references `manifest.project_name`, `manifest.camera`, `manifest.simulation.duration_seconds` -- none of which exist in the actual KFSManifest model.

**Root cause:** Despite Fix B (sharing file content), the retry cycles restarted from scratch. The content sharing only works within a single build pass, but cumulative_files resets between retries because it's local to `builder_node()`.

**Fix needed:** Either persist cumulative file content across retries (in state), or increase `_MAX_TOKENS` from 4096 to allow richer code generation in a single pass.

### Important: Test stubs cause pytest collection errors

All test files are 1-line stubs, causing pytest collection to fail with "11 errors during collection". This is the verify layer 1 failure.

**Same root cause as above** -- stubs not overwritten.

### Important: _MAX_TOKENS = 4096 too low

For a task that needs to generate both models and tests (T2 generated kfs_core/models.py at 82 lines + test file), 4096 tokens is marginal. The LLM may truncate or simplify.

**Fix needed:** Increase to 8192 for the build stage.
